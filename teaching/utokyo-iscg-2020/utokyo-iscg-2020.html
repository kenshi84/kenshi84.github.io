課題

Assignments

  • ライブラリ
  • Libraries
    • legacygl.js (repository, website)
    • glMatrix
  • 提出方法
    • WebGLで実装したものをWeb上にアップロードし、そのURLをTAにメールで送ること。 Glitchを使うのが簡単だが、自前サーバ等を使っても良い。
    • HTMLページ上に簡単な内容説明や考察等を含めること。
    • legacygl.js以外のWebGLライブラリを用いても良い。
    • C++等で実装しても良いが、一般的な環境でビルド・実行できるようにすること。上記と同様のレポートをソース・バイナリと合わせてzipファイルとして提出すること。
    • 課題は大まかな形で出題するので、具体的に何をどう実装するかは各人の興味と力量に応じて決めること。実装内容のクオリティは成績評価に反映される。
    • 基本課題の締切は、指定日の23:59:59とする。
    • 氏名及び学籍番号（学生証番号）、課題URLをメール件名・本文・課題ページ内のいずれかに明記すること。
    • 課題提出メールには必ず返信をする。一週間経っても返信が来ない場合は適切に受理されていない可能性があるので、TAまで確認すること。
    • 提出期限までは何度でも再提出可能（新しい方のみ採点対象とする）。
    • 不明な点があればすぐ相談すること。
  • Requirements
    • Implement the solution using WebGL, upload it to the web, and send its URL to the TA via e-mail. Use of Glitch is recommended, but other ways (e.g. your own server) are also allowed.
    • Include brief description/discussion/etc in the HTML page.
    • Other WebGL libraries than legacygl.js can also be used.
    • Implementation with other languages such as C++ is allowed, provided that it compiles and runs under typical computing environments. In this case, some written report as noted above should be included along with the source code and binary in a single ZIP file.
    • Assignments are described in less specific ways, so it is up to you to implement what features in which ways. The quality and creativity in the submitted implementations will be reflected in the final grade.
    • Deadlines of Basic assigments are on 23:59:59 of the specified date.
    • State your name, student number, your assignemnt website URL, etc. in either the email title, message, or in the website itself.
    • Every submission will get a reply from the TA. No reply within a week likely means your submission wasn't received properly, so contact the TA.
    • You can update your submission any number of times before the deadline. The latest one gets evaluated.
    • If anything is unclear, don't hesitate to contact the TA or the lecturers.
